# startcpp

`startcpp` is a small Bash script with no extra dependencies beyond the usual build tools. Its job is simple: spin up a modern C++23 workspace in a few seconds, already set up for **Clang**, **Ninja**, **CMake**, and **Modules**.

It takes care of the boring first steps for you: it creates the project layout, wires in **CPM** for dependencies, tries system packages first before falling back to GitHub when needed, enables `ccache` if it is available, and links `compile_commands.json` straight into the project root so tools like `clangd` can pick it up right away.

---

## What it creates

- A **C++23** project
- Built-in support for **C++ Modules**
- A **CMake + Ninja** build setup
- Automatic **ccache** integration
- `compile_commands.json` linked at the project root
- A small logger module (`Log.cppm`) using `std::source_location`
- Ready-made templates for:
  - a command-line app
  - **SDL2**
  - **SDL3**
  - **Raylib**

---

## What it solves

The idea is to remove the repetitive setup work that usually happens at the start of a new project:

- creating folders by hand
- writing the initial `CMakeLists.txt`
- setting up modules
- adding compiler flags
- putting together a basic logger
- making the LSP work without extra steps

So instead of starting from an empty folder and patching things together later, you begin with a project that already has a clean structure.

---

## Requirements

Before using it, make sure you have:

- **Clang** with C++23 support
- **CMake 3.28 or newer**
- **Ninja**
- **ccache** *(optional, but recommended)*

---

## Installation

Clone the repository and place the script somewhere in your `PATH`:

```bash
git clone https://github.com/V1niciosLins/startcpp.git
cd startcpp
chmod +x startcpp
mv startcpp ~/.local/bin/
```

---

## Usage

### Default project

```bash
startcpp my_project
```

### Graphics / game templates

```bash
startcpp my_project --sdl2
startcpp my_project --sdl3
startcpp my_project --raylib
```

### List available options

```bash
startcpp --list
```

## What each template generates

### Default template

A simple C++23 starter with:

- `main.cpp`
- the `Log` module
- a basic hello-world entry point
- a structure that is ready to grow

---

### SDL2

Creates a project with:

- SDL2 window initialization
- a basic event loop
- integrated logging

---

### SDL3

Creates a project using SDL3’s modern callback model:

- `SDL_AppInit`
- `SDL_AppEvent`
- `SDL_AppIterate`
- `SDL_AppQuit`

---

### Raylib

Creates a small graphical starter with:

- a window
- a render loop
- `SetTargetFPS`
- a minimal visual example

## Generated structure

```text
├── build/                 # Ninja/CMake build artifacts
├── compile_commands.json  # link to the compilation database
├── CMakeLists.txt         # C++23 + modules build setup
├── .clang-format          # modern formatting rules
├── .gitattributes         # Linguist rules
├── .gitignore             # project exclusions
├── include/
│   └── Log.cppm           # logging module
└── src/
    └── main.cpp           # entry point
```

After generating the project:

```bash
ninja -C build
./build/bin/<project_name>
```

## Highlights

### C++23 Modules

The project is ready for `.cppm` files from the start, which makes module-based development much easier.

---

### CPM with fallback

Dependency resolution tries the system first and only downloads from GitHub if it really needs to.

---

### `compile_commands.json` ready to use

The script links the compilation database into the project root, which is very convenient for:

- `clangd`
- terminal editors
- autocomplete
- code navigation

---

### `ccache`

If `ccache` is installed, it gets wired in automatically to speed up rebuilds.

---

### Built-in logger

The project includes a lightweight logging module with:

- log level
- file/line/function information through `std::source_location`
- optional file logging

---

## Project philosophy

`startcpp` is not trying to hide CMake behind a giant abstraction layer.

The goal is give you a modern, clean starting point that you can expand however you want.

You keep full control over:

- build
- dependencies
- folder layout
- compiler flags
- project architecture

The script just removes the repetitive setup work that usually eats the first few minutes of every new project.

---

## Notes

- The logger and templates are starting points, not rigid rules.
- Everything generated by the script can be changed freely.
- The project is mainly designed for Linux and Clang-based toolchains.
